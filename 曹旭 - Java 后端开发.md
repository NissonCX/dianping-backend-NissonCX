## 项目经历

### 虎溪锐评 (后端开发)



**时间**: 2025-07 至 2025-08 **技术栈**: Spring Boot + MyBatis-Plus + MySQL + Redis (缓存&分布式锁) + RabbitMQ + Lua + JWT + Nginx + Docker 



**项目简介**: 仿大众点评的本地生活服务平台，支持商户浏览、探店笔记、点赞收藏、优惠券秒杀等核心功能。重点实现高并发场景下的优惠券秒杀系统，通过多种分布式技术保障数据一致性与系统稳定性 。



**核心职责与成果**:

- 

  **防止重复领取**: 实现“一人一笔”业务约束，采用缓存预检+锁串行+唯一索引 (user_id + voucher_id)，从底层杜绝重复领取 。

  

  

- 

  **解决超卖问题**: 采用 Redis 预减库存 + Lua 原子脚本实现库存扣减，确保高并发下库存操作的原子性 。

  

  

- 

  **异步削峰**: 结合 RabbitMQ 异步下单，降低数据库压力，提升系统吞吐量；在创建订单前进行二次库存校验，防止缓存与数据库不一致导致超卖 。

  

  

- 

  **分布式锁**: 针对用户重复点击场景，基于 Redis+SETNX+过期时间实现分布式锁，确保同一用户请求串行执行，保障幂等性 。

  

  

- 

  **高性能缓存**: 使用 Redis 缓存热门商户与探店笔记，QPS 提升3倍；采用“逻辑过期”策略解决缓存雪崩，结合互斥锁重建防止缓存击穿 。

  

  

- 

  **限流与防护**: 通过 Redis + Lua 实现令牌桶算法，对秒杀接口进行 QPS 限流 (如1000/s)，防止恶意刷单 。

  

  

- 

  **全链路压测**: 使用 JMeter 模拟5000并发用户秒杀，系统稳定运行，无超卖、无重复领取，平均响应时间 <150ms 。

  

  

### 吃了么 (后端开发)



**时间**: 2025-06 至 2025-07 **技术栈**: Spring Boot + MyBatis + MySQL + Redis (缓存&分布式锁) + WebSocket + JWT + Nginx 



**项目简介**: 仿美团外卖的外卖订餐系统，涵盖用户端 (小程序风格 Web 界面) 与商家管理后台，支持菜品浏览、购物车管理、下单、支付模拟、实时状态推送及数据统计等功能 。



**核心职责与成果**:

- 

  **前后端分离架构**: 采用 RESTful API 规范设计接口，使用 Swagger 自动生成文档，提升协作效率 。

  

  

- 

  **高性能缓存策略**: 将高频访问的菜品分类与套餐数据缓存至 Redis，接口平均响应时间从 320ms 降至 80ms 。

  

  

- 

  **订单状态实时同步**: 基于 WebSocket 实现用户端与商家端的订单状态 (接单、派送、完成) 实时推送 。

  

  

- 

  **安全与权限**: 用户端采用 JWT + 拦截器实现无状态认证；管理端基于 RBAC 模型实现角色-菜单-接口三级权限校验 。

  

  

- 

  **数据一致性**: 在修改菜品状态或库存时，采用 Redis 缓存与数据库双写一致性策略 (先更新 DB，再删除缓存)，降低脏读风险 。

  

  

- 

  **数据可视化**: 基于 MySQL 聚合查询 + 定时任务 (@Scheduled)，实现营业额、订单量等关键指标日报统计并在后台展示 。
